Project Spotlight:
Kubernetes Homelab

This homelab is the production environment for my day-to-day tools and the place I experiment with new platform ideas. It runs as a two-node k3s cluster, embraces GitOps, and powers everything from media streaming to a freshly reworked llama.cpp + LibreChat AI stack. Every service is declarative, version-controlled, and deployed automatically through ArgoCD.

The mandate is data freedom with a sustainable ops story. That means evolving the cluster toward high availability, keeping backups first-class, and carving out room to test tomorrow's workflows without risking today's uptime.

Architecture Snapshot

Lightweight Core

k3s keeps the cluster lean without dropping production features. The control plane stays compact while MetalLB exposes LoadBalancer services, HAProxy terminates TLS, and containerd spreads workloads across the two nodes.

GitOps All the Way

ArgoCD owns desired state with the app-of-apps pattern. Kustomize bases and overlays separate shared configuration from environment tweaks, and the platform/test suite validates nodes and ingress paths before production apps sync.

Resilient Storage

Longhorn replicates persistent volumes across both nodes. Hourly snapshots and nightly NAS backups protect everything from Plex metadata to password vaults, keeping recovery a Git sync and volume restore away.

Simple Network Flow

Traffic flows Cloudflare → HAProxy → services, split by ingress classes for public vs. restricted apps. cert-manager keeps certificates fresh without manual effort.

Operational Rhythm

Day-to-Day Flow

  • Edit manifests ➜ push ➜ ArgoCD's app-of-apps reconciles drift within minutes.
  • Smoke-test new nodes and ingress paths with platform/test before syncing live apps.
  • Helper scripts bootstrap k3s, check certificates, and join additional nodes.

Why It Works

  • app-of-apps structure keeps platform services, apps, and infrastructure isolated.
  • Pod disruption budgets, HPAs, and anti-affinity rules protect the workloads that matter most.
  • Ingress templates, ADRs, and documentation make change history explicit.

What Runs Here

Media & Entertainment

Plex, qBittorrent behind Gluetun, and a full *arr stack keep the library fresh.

Productivity & Security

Vaultwarden, Nextcloud + Collabora, and Immich replace third-party SaaS.

AI & Development

llama.cpp handles inference, LibreChat runs across replicas, and MongoDB + Qdrant back the conversational memory.

Platform Essentials

ArgoCD, cert-manager, HAProxy, MetalLB, and Longhorn form the backbone services.

High Availability & Safeguards

Cluster Layout

Two-node k3s cluster, single writer control plane.

Critical Replicas

HAProxy, CoreDNS, cert-manager, Vaultwarden, LibreChat, and the portfolio run multi-replica.

Data Protection

Longhorn snapshots hourly, backups nightly to NAS.

Everything is built to fail smart: pods respect anti-affinity, storage self-heals, and automation keeps drift from sneaking in.

What I'm Learning

Platform Thinking

Modeling infrastructure as reusable building blocks—GitOps apps, ingress templates, ADRs, and runbooks—keeps change intentional even while experimenting.

HA Playbooks

Rolling out pod disruption budgets, HPAs, and anti-affinity policies workload by workload while validating restore drills builds repeatable failover practices.

Security in Motion

Network policies, certificate automation, and namespaced secrets enforce safe defaults that are expanding as more services move to multi-replica mode.

Try It Yourself

  1. Clone the repo and review infrastructure/k3s for hardware tweaks.
  2. Run scripts/install-k3s.sh on your node to bootstrap k3s.
  3. Apply the ArgoCD manifests, then deploy the root app.
  4. Use the workloads in platform/test to verify ingress and certificates.
  5. Let ArgoCD sync the production overlays and tune them for your network.

The docs folder captures ADRs, architecture notes, and templates to remix the setup for your environment when you're ready to move beyond the defaults.
