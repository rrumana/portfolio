Project Spotlight:
This Website

Welcome to my homelab! It may not seem like it, but I am self-hosting this website and everything on it, including the Game of Life simulation. This website serves as a personal portfolio and a playground for my projects, where I can showcase my work and share my journey as a software engineer.

My goal with this project is to create a centralized place for all of my hobby projects, thoughts, and life updates that will grow as I do. A year ago I had absolutely no idea how to build a website, and since then I have learned so much about web development.

I hope you find this website—and the explanation for why I decided to build and host it this way—to be both informative and entertaining!

Homelab Hardware

My homelab is built from a combination of repurposed and custom-built hardware. It features dedicated servers, network-attached storage, and a high-performance router, all running on energy-efficient components. This setup allows me to self-host my website, run experiments, and deploy personal projects without relying on third-party services.

The hardware is maintained in a controlled environment to ensure optimal performance and uptime, and I continually upgrade components as needed.

Software Stack

This website is built using a modern web development stack. The backend is written in Rust using the Axum framework, and it serves both static content and API endpoints. The front end is crafted with HTML, CSS (using SCSS for styling), and JavaScript. Additionally, WebAssembly is used to power the interactive Game of Life simulation.

For version control, continuous integration, and deployment, I use GitHub along with various DevOps tools to streamline my workflow.

Deployment & DevOps

The website is deployed using Docker and Docker Compose, allowing for easy scalability and management. I containerize the backend, WebAssembly modules, and static assets so that the entire application can run in a consistent environment across different platforms.

Monitoring, logging, and automated backups are integrated into the deployment process. My DevOps pipeline includes automated testing and continuous integration, ensuring that every update is stable and reliable before it goes live.

Future Work

I plan to expand this website with new interactive features, integrate more advanced analytics, and experiment with new web technologies. Future updates may include real-time collaboration tools, enhanced visualization of data, and deeper integration of WebAssembly for performance-critical tasks.

In addition, I'll explore further optimizations for the Game of Life simulation and possibly extend its functionality to cover other cellular automata and complex systems.
