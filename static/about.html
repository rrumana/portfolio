Project Spotlight:
This Website

Welcome to my homelab! It may not seem like it, but I am self-hosting this website and everything on it, including the Game of Life simulation. This website serves as a personal portfolio and a playground for my projects, where I can showcase my work and share my journey as a software engineer.

My goal with this project is to create a centralized place for all of my hobby projects, thoughts, and life updates that will grow as I do. A year ago I had absolutely no idea how to build a website, and since then I have learned so much about web development.

I hope you find this website—and the explanation for why I decided to build and host it this way—to be both informative and entertaining!

Homelab Hardware

This website is hosted on a Minisforum Elitemini AI370 mini-PC featuring a powerful 12-core AMD Ryzen AI 370HX CPU and running Arch Linux. Data integrity and redundancy are ensured through a robust BTRFS RAID 1 filesystem, complemented by frequent snapshots and backups to a dedicated cold spare drive. Networking is secured by a Partaker 1U firewall appliance running OPNsense on an Intel Atom C3958, effectively protecting and managing internal traffic through a centrally managed switch. In addition to serving the website, this multifunctional server operates around 37 Docker containers, functioning as a media center, phone backup solution, personal cloud instance, and primary development workstation.

Software Stack

The website leverages Rust’s performant Axum framework to serve static HTML and SCSS-generated stylesheets efficiently. Interactivity, including animations and transitions, is achieved through minimal JavaScript usage. Performance-intensive components, particularly the interactive Game of Life simulation, are developed in Rust and compiled to WebAssembly (WASM), effectively utilizing client-side resources for computation. The host system uses Hyprland as a lightweight Wayland compositor, optimizing resource usage and enhancing responsiveness. Development was done on a Neovim setup I made my self.

Deployment & DevOps

Docker containerization ensures reliable and reproducible deployments, streamlining updates and minimizing downtime. Nginx is configured as a reverse proxy, securely bridging containerized applications to the public internet. Domain management and DNS services provided by Cloudflare deliver additional layers of security, including DDoS protection, caching, and traffic filtering. Further, legitimate traffic passing Cloudflare’s checks is validated by a self-managed proxy manager. Internal observability is provided by BTOP for system resource monitoring and Homarr dashboard, which centralizes access to internal services behind WireGuard, ensuring secure, convenient, and remote management capabilities.

Future Work

Future expansions include migrating toward a clustered computing environment for increased redundancy, scalability, and fault tolerance—reducing disruption during power outages, maintenance, or updates. Additionally, consideration is being given to transitioning the site from primarily static content delivery to dynamic, feature-rich experiences. Potential adoption of JavaScript frameworks such as React or Rust-based web technologies like Yew could provide advanced logging, data management, and enhanced user interactions while maintaining stack efficiency and consistency.

In addition to those service improvments I plan to further explore optimizations for the Game of Life simulation and possibly extend its functionality to cover reveral and other complex behavior.
